# 1.0.0 Release Notes

## Scope

`1.0.0` is the first tagged repo state intended to be operable as a standalone Rust Discord streaming transport with a high-level CLI.

## Included capabilities

- daemon HTTP control plane
- Discord session and voice/media negotiation
- WebRTC publisher path
- Unix-socket ingest for H.264 and Opus
- `play` command for:
  - local files
  - X11 + Pulse capture
- Docker packaging for the Neko integration path
- operator documentation for the daemon and CLI surface
- token-hygiene fix for the container entrypoint so Discord auth is not exposed in process arguments

## Validation completed

- `cargo test -p cli`
- `cargo build -p daemon -p cli --features webrtc-media`
- `docker compose config` in the Neko integration repo
- rebuilt Compose smoke test for the X11 + Pulse path
- verified `/v1/health` and `/v1/media/health`
- verified the active container runs `discord-rs-streamer ... play ...`
- verified the Discord token is not present in container process arguments

## Operational expectations

- For the Rust streaming path, use a stock Neko image and mount runtime configuration around it.
- Keep live Discord tokens in local `.env` files or container env injection only.
- Do not commit release zips, local logs, or operator scratch files.
