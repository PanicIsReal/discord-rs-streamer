# 1.0.0 Release Candidate Notes

## Scope

This release candidate is the first repo state intended to be operable as a standalone Rust Discord streaming transport with a high-level CLI.

## Included capabilities

- daemon HTTP control plane
- Discord session and voice/media negotiation
- WebRTC publisher path
- Unix-socket ingest for H.264 and Opus
- `play` command for:
  - local files
  - X11 + Pulse capture
- Docker packaging for the Neko integration path

## Operational expectations

- For the Rust streaming path, use a stock Neko image and mount runtime configuration around it.
- Keep live Discord tokens in local `.env` files or container env injection only.
- Do not commit release zips, local logs, or operator scratch files.

## Suggested validation before a formal tag

- smoke test file playback through `play --input`
- smoke test X11 + Pulse playback through Docker Compose
- verify `/v1/health` and `/v1/media/health`
- verify stream startup and reconnect behavior in Discord clients
